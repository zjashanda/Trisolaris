# 外部项目验证模式提取记录

## 来源仓库

- `mars-moon`：杜亚窗帘类离线语音项目，重点是需求结构化、workspace bundle、设置窗口/模式状态机、同义词与负例。
- `mars-belt`：MarsPlatform 打包和固件验证项目，重点是平台支持矩阵、边界/中值包、配置断言、控制变量复测和重启异常优先级。
- `CSK5062 小度风扇`：本仓库项目，重点是语音注册、命令词/唤醒词学习、模板满、配置保存和聚合断言。
- `CSK3022 好太太晾衣机`：本仓库项目，重点是主动/被动协议闭环、play id、音量探测、语音关闭状态机和台架握手仿真。

## 抽取到验证池的关键模式

### mars-moon

- 需求不是命令词平铺表，而是“状态 + 触发 + 反馈 + 持久化”的结构。
- 用例生成分成 `BOOT / BASE / VOL / UART / POWER / FACTORY / WAKEWORD / WORKMODE / CURTAINMODE / CURTAIN / PHRASE / SELECTOR / CTRL` 等模块。
- 设置类命令必须走：唤醒 -> 进入设置 -> 目标值 -> 设置成功 -> 后置行为验证。
- 设置窗口内普通控制词在部分项目里可能允许执行，不能默认当无效词；真正要验证是否污染后续设置。
- 模式敏感短语必须自建前置模式，不能借用上一条 case 残留状态。
- 同义词可以分组，但报告要展开每个词。
- 每次重启后必须恢复日志等级并确认日志恢复。
- 主动重启/主动上下电不算异常；正常执行中的被动重启才算异常。

### mars-belt

- 平台打包项必须先查支持矩阵，不能擅自换产品/模块/语言/版本。
- 只测当前产品支持或可编辑项，不支持项透明披露，不计 FAIL。
- 配置断言和运行态验证分层；配置没入包是平台/模板映射问题，不是运行态功能失败。
- 数值/枚举按左边界、中值、右边界组合；字符串验证一次；布尔覆盖 true/false。
- 依赖项同包验证，组合包失败后必须控制变量复测。
- 非预期重启优先级最高，不得被后续 PASS 吞掉。
- pretest 失败立即停止，不允许跳过。

### 小度风扇

- 语音注册必须验证学习成功后的实际可用性。
- 删除必须先证明已学习词可用，删除后证明不可用。
- 冲突词要验证拒绝学习，并确认原功能不被篡改。
- 模板满必须用用例内主动填满，不能只看启动计数。
- 保存类链路必须等待保存闭环后重启。

### 好太太晾衣机

- 主动协议上报 MCU 后，本地状态经常要等 MCU 被动回包才能改变。
- 语音开关不能套固定回包；当前项目经探测收敛为 `0x0016 -> 0x0036`、`0x0017 -> 0x0037`。
- 被动 `0x0012` 是 MCU 直接关闭语音入口，不是主动打开语音回包。
- `0x008C` 按“设置失败”播报，断言 `play id 127`。
- 音量默认值和总档位必须用单边/双边边界探测，不直接用 raw 编码。
- 跨串口计时可能偏移，超时优先用日志口同源标记。

## 合并原则

- 以上内容已拆入 `validation-pool` 各模块。
- 任何项目特定码值只作为模块中的已知变体示例，不作为新项目默认断言。
- 新需求优先匹配模块和变体；旧项目文档只能作为实现参考。
