# 主动上报与 MCU 回包闭环

## 适用需求特征

- 语音命令先主动上报 MCU，模块本地状态或播报需要等待 MCU 回包后才变化。
- 词表有“主动发送”和“MCU 回复”两列。

## 变体维度

- 回同码确认。
- 回语义对应码确认。
- 主动命令只上报，不改变本地状态。
- Excel 行对照与播报文案冲突，需要探测确认。

## 需求解析字段

- 主动协议、候选 MCU 回包、配置刷新字段、播报字段、后续业务效果。

## 验证方案模板

1. 执行主动命令并捕获主动协议。
2. 按需求/词表注入 MCU 回包。
3. 观察本地配置、play id、业务能力是否改变。
4. 如回包不生效，使用最小探测法测试候选回包。
5. 将最终闭环规则写回项目用例。

## 用例模板

- `CLOSED-LOOP-STATE-001`
- `CLOSED-LOOP-PROBE-001`
- `CLOSED-LOOP-RECOVER-001`

## 断言与证据

- 不得在 MCU 回包缺失时断言本地副作用失败。
- 若回包导致状态相反，以配置刷新和后续业务效果为准。
- 好太太当前变体：主动关闭 `0x0016 -> 0x0036`；主动打开 `0x0017 -> 0x0037`；被动 `0x0012` 是 MCU 直接关闭语音入口。

## 执行器映射

- 好太太：`run_htt_handshake_formal_suite.py`、`run_htt_voice_switch_state_machine_probe.py`。

## 回灌规则

- 同一功能在不同项目回包可能不同；只回灌“需要闭环和探测”的方法，不把好太太码值当全局规则。
