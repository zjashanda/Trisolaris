# 设置窗口、模式状态和普通词干扰

## 适用需求特征

- 需求包含“进入设置 X 模式后，再说目标值完成设置”。
- 存在工作模式、窗帘模式、唤醒词候选、专有词互斥或设置窗口超时。

## 变体维度

- 设置窗口内普通控制词允许执行并保持窗口。
- 设置窗口内普通控制词应阻断。
- 模式词在设置外无效。
- 模式切换后立即影响后续命令。

## 需求解析字段

- 设置入口词、候选目标值、窗口超时、普通词处理、互斥词组、默认模式。

## 验证方案模板

1. 默认状态验证。
2. 设置入口进入。
3. 候选值设置成功。
4. 后续行为验证状态生效。
5. 设置外候选词反例。
6. 设置窗口内普通词干扰验证。
7. 超时退出和恢复路径。

## 用例模板

- `SETTING-ENTRY-001`
- `SETTING-VALUE-001`
- `SETTING-OUTSIDE-BLOCK-001`
- `SETTING-COMMON-WORD-INTERFERENCE-001`
- `MODE-SWITCH-VERIFY-001`
- `MODE-MUTEX-NEGATIVE-001`

## 断言与证据

- 设置成功播报不是最终结论，必须补后置行为验证。
- 模式敏感短语不能借上一条 case 残留状态，每条用例自建前置。
- 同义词组可归组，但报告中必须展开每个具体词。

## 执行器映射

- mars-moon：`dooya_case_builder.py`、`dooya_voice_runner.py`。

## 回灌规则

- 新设置窗口规则必须写清楚普通词是否允许、窗口是否保持、超时如何退出。
