# 语音开关和受限状态机

## 适用需求特征

- 需求包含关闭语音、打开语音、关闭后仍可唤醒、普通命令受限、受限窗口或提示音。

## 变体维度

- 关闭后完全不响应。
- 关闭后仍可唤醒，但只允许“打开语音”。
- 当前会话未超时与已超时后受限窗口不同。
- 打开语音是否同时恢复播报。
- 主动开关是否需要 MCU 回包闭环。

## 需求解析字段

- 关闭命令、打开命令、主动协议、被动协议、受限提示、允许命令、受限超时、普通会话超时。

## 验证方案模板

1. 默认语音可用。
2. 主动关闭语音闭环。
3. 关闭后普通命令阻断。
4. 关闭后当前会话未超时分支。
5. 关闭并等待超时后受限分支。
6. 受限窗口首次提示/后续不重复。
7. 打开语音恢复业务和正常超时。

## 用例模板

- `VOICE-OFF-BLOCK-001`
- `VOICE-ON-RECOVER-001`
- `VOICE-OFF-CURRENT-SESSION-25S`
- `VOICE-OFF-DING-ONCE`
- `VOICE-OFF-AFTER-TIMEOUT-10S`
- `VOICE-ON-RECOVER-25S`

## 断言与证据

- 关闭后允许唤醒不等于业务恢复；业务协议必须继续阻断。
- 当前会话 25s 和受限 10s 要分支断言。
- 好太太当前变体：主动关闭 `0x0016 -> 0x0036`，主动打开 `0x0017 -> 0x0037`，`0x0036` 不能作为打开语音回包。

## 执行器映射

- 好太太：`run_htt_voice_switch_state_machine_probe.py`、`run_htt_handshake_formal_suite.py`。

## 回灌规则

- 其他项目若语音关闭逻辑不同，新增变体，不复用好太太 10s/25s 断言。
