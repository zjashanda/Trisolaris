# 好太太语音开关状态机重设计用例 v1

## 设计口径

依据用户澄清和 `项目需求/好太太晾衣机/语音关闭逻辑.png`，`G-07` 不再按“关闭语音后任意唤醒均应 10s 超时”断言，而是拆成以下状态：

1. 关闭语音后，如果仍处于上一轮 25s 休眠倒计时内，再次唤醒仍走 25s 超时。
2. 关闭语音后，当前 25s 对话已超时并进入休眠后，再次唤醒才走 10s 受限窗口。
3. 关闭语音后第一次受限唤醒有“咚”提示，后续在受限窗口内再次唤醒不应重复“咚”。
4. 语音功能打开后，应退出受限态，业务恢复，超时重新回到 25s。

## 用例清单

| 用例 ID | 目的 | 前置/动作 | 期望 |
| --- | --- | --- | --- |
| `VOICE-OFF-CURRENT-SESSION-25S` | 验证关闭语音后仍在上一轮 25s 倒计时内的唤醒超时 | 恢复默认 -> 被动 `0x0012` 关闭语音 -> 不等 25s 结束直接唤醒 | 有受限唤醒，普通业务阻断；`Wakeup/0x0001/play id 77 -> TIME_OUT/MODE=0` 约 25s |
| `VOICE-OFF-DING-ONCE` | 验证关闭语音后当前受限窗口内只首次“咚” | 恢复默认 -> 被动 `0x0012` -> 连续两次唤醒 | 关闭语音提示存在；受限唤醒“咚”类提示只出现 1 次，后续唤醒不重复 |
| `VOICE-OFF-AFTER-TIMEOUT-10S` | 验证关闭语音并等上一轮 25s 超时后再次唤醒走 10s | 恢复默认 -> 被动 `0x0012` -> 等待上一轮 25s 超时 -> 再唤醒 | 受限唤醒后约 10s 进入 `TIME_OUT/MODE=0` |
| `VOICE-ON-RECOVER-25S` | 验证打开语音后恢复正常会话 | 恢复默认 -> 主动 `0x0016` 关闭语音并回 `0x0036` -> 等待上一轮 25s 超时 -> 唤醒 -> 语音功能打开 -> 夹具回 `0x0037` -> 再唤醒/打开照明 | 应下发 `0x0017`，收到 MCU 回 `0x0037` 后后续普通业务应恢复 `0x0009`，正常会话超时约 25s；实测 `0x0036` 对应关闭语音状态，不能作为开语音回包 |

## 判定原则

- 最终 FAIL 只保留固件行为或需求口径不一致。
- 若实测 25s 出现在 `VOICE-OFF-CURRENT-SESSION-25S`，应判 PASS，不再归入旧 `G-07` FAIL。
- 只有 `VOICE-OFF-AFTER-TIMEOUT-10S` 实测不为 10s，才保留新的 G-07 类 FAIL。
