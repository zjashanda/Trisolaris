## 2. 本轮新增证据

- 新增专项探针：
  - `deliverables/csk3022_htt_clothes_airer/reports/20260423_222404_htt_voice_restricted_probe_r1/summary.md`
- 原始结果：
  - `result/csk3022_htt_clothes_airer/20260423_222404_htt_voice_restricted_probe_r1/`

本轮新增结论：

1. `关闭语音受限窗口=10s`：`FAIL`
   - 被动 `0x0012` 关闭语音后，再次唤醒进入受限态；
   - 真实观测到：
     - `受限唤醒 -> TIME_OUT = 25.881s`
     - `受限唤醒 -> MODE=0 = 25.881s`
     - `受限响应结束 -> TIME_OUT = 25.185s`
     - `受限响应结束 -> MODE=0 = 25.185s`
   - 无论按“受限唤醒标记”还是按“受限响应结束”计算，都显著大于需求 `10s`，实际更接近普通会话 `25s`。
2. 受限态 `play id 77 / 123`：`PASS`
   - 被动 `0x0012` 后：
     - 再次唤醒的受限提示为 `play id 77`
     - 在受限态说普通命令“打开照明”时，提示为 `play id 123`
   - 同一步骤中仍无 `0x0009` 下发，说明 `123` 对应的是“语音已关闭提示”而不是业务执行结果。
