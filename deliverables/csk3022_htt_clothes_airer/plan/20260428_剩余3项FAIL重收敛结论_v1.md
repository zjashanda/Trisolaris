## 2. 语音开关回包口径

Excel 行内存在容易误读的主动/被动列错位现象，因此本轮同时做了探测验证：

| 探测项 | 实测状态变化 | 结论 |
| --- | --- | --- |
| `0x0017 -> 0x0036` | 收到 `0x0036` 后日志仍刷新 `voice=0`，播报 `play id 24`，后续普通业务仍受限 | `0x0036` 不能作为开语音回包 |
| `0x0017 -> 0x0037` | 收到 `0x0037` 后日志刷新 `voice=1`，播报 `play id 25`，后续“打开照明”下发 `0x0009` | `0x0037` 是开语音恢复回包 |
| `0x0016 -> 0x0036` | 收到 `0x0036` 后进入 `voice=0` 受限态 | `0x0036` 是主动关闭语音闭环回包 |
| 被动 `0x0012` | 直接进入 `voice=0` 受限态，播报 `65` | `0x0012` 是 MCU 直接关闭语音入口，不是主动关闭词条的唯一回包 |

因此，正式用例当前按以下规则执行：

- 主动“语音功能关闭”：`0x0016 -> 0x0036`
- 主动“语音功能打开”：`0x0017 -> 0x0037`
- MCU 直接关闭语音：被动 `0x0012`
