## 2. 最新验证逻辑

### 2.1 MCU 回包闭环

好太太项目中，多数主动命令只是 CSK 上报 MCU；如果要断言本地状态、副作用或播报，必须补齐 MCU 被动回包后再判断。

- 主动音量：`0x0041` / `0x0042` 等需要回同码被动协议后，再读取本地音量变化。
- 主动关闭语音：`0x0016 -> 0x0036`，闭环后进入 `voice=0` 受限态。
- 主动打开语音：`0x0017 -> 0x0037`，闭环后恢复 `voice=1` 和普通业务。
- 被动 `0x0012` 是 MCU 直接关闭语音的独立入口，不能替代主动开语音回包。

如果 Excel 同行回包和播报文案/配置刷新冲突，必须用最小探测链路验证哪个回包真正改变本地状态，再写回用例和断言。

### 2.2 语音开关状态机

最新口径按用户澄清拆分：

1. 关语音后，如果当前 25s 对话尚未超时，再唤醒仍处在当前会话窗口内，超时约 25s。
2. 关语音后，当前对话已经超时，再唤醒才进入受限窗口，超时约 10s。
3. 关语音后的受限唤醒首次有“咚”提示，受限窗口内后续唤醒不重复“咚”。
4. 开语音后退出受限态，业务协议恢复，正常会话超时回到约 25s。

对应专项用例：

- `deliverables/csk3022_htt_clothes_airer/cases/20260427_语音开关状态机重设计用例_v1.md`
- `tools/debug/run_htt_voice_switch_state_machine_probe.py`

### 2.3 `0x008C` 设置失败

需求文档中的“新增被动播报：`0x008C`，设置失败，未播报”按产品澄清解释为：

- 新增一个被动播报协议；
- 收到 `0x008C` 后需要播报“设置失败”；
- 断言期望 `play id 127`；
- 不再按“不允许播报”判断。

### 2.4 TTS 别名收敛

单个 TTS 短语稳定打到同表邻近意图时，先用同一需求行的官方别名探测。别名通过时，更新用例短语，不把 TTS 选择问题保留为最终固件 FAIL。

本轮已收敛：

- `FULL-WARM-UP-001` 使用 `调暖一点`
- `FULL-WARM-MAX-001` 使用 `调到最暖`

### 2.5 数值和默认值

- 验证固件默认值前，如果存在 `config.clear` 或等价全配置清除能力，必须先清配置再烧录，确保默认值来自固件。
- 默认音量不能用启动日志中的 raw 编码直接反推，必须用单边探测到边界，再用双边边界计算真实档位。
- 唤醒超时以日志口同源 `MODE=1/Wakeup -> TIME_OUT/MODE=0` 计时；协议口 `0x0001` 只作旁证，避免跨串口时间戳偏移。

## 4. 反例和异常覆盖边界

当前 96 条包含功能级反例、状态异常、边界和恢复链路：

- 未唤醒直说命令阻断；
- 超时后未重唤醒命令阻断；
- 关闭语音后普通命令受限；
- 语音关闭/打开状态机；
- `0x008C` 设置失败播报；
- 播报关闭后不产生额外播报；
- 默认音量、音量边界、掉电保存。

当前未纳入完整鲁棒性专项：

- 非法协议帧、错校验、错长度、错地址；
- MCU 不回包、延迟回包、回错协议；
- 串口断连、半包、粘包；
- 强噪声、多人声、误唤醒干扰；
- 写配置中断电、烧录中断电；
- 长稳压力循环。

1. 固件不满足需求；
2. 需求本身错误或矛盾。
